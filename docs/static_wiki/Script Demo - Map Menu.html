When using the complete function of the
  • map,
  • sp,
  • dmsp,
  • ffa,
  • insta and
  • capture
aliases you always use an up-to-date list of maps (=*.ogz files).
This means they will auto-complete even your just saved mapname.

Beware of menu code differences between the 2006-09-12 (water) and the 2006-12-04 (gui) releases of Sauerbraten.
The from 2D to 3D paradigm shift forces us to go dual track here, for a while!
Always check your local documentation when in doubt!

The menus do not auto-update themselves;
these lists are editorial only - meaning they are a selection made by a human.
You can easily add your favourite creations, downloads and coop-edit maps by adding them to the map name lists in data/menus.cfg.
Look for the line reading
// 2D style
newmenu maps

// 3D style
newgui maps

2D style modification

and after the last line containing menuitem before any new newmenu:
add your own menuitem like so
menuitem "my favourites.." "showmenu favmaps"

newmenu favmaps
genmapitems "my_fav_001 work001 work002 work003"

3D style modification

insert after the line with rpgmaps = " (line 49 in 2006-12-04 release)
personalmaps = "my_personal_favourite_map_nr_1 my_personal_favourite_map_nr_2 my_personal_favourite_map_nr_3"
and inside the newgui maps block - at appropriate position
guitab personal
  guilist [
    guilist [ genmapitems $personalmaps ]
    showmapshot
  ]

Assuming the maps you entered truely exist in packages/base your menu modification will work.
