Basic Uses


Each map will first load default_map_settings.cfg
and then any package CFG and last the specific map CFG.
So a map called metl4.ogz will call metl4.cfg.

The first need for such a specific map configuration file will most likely arise when you want to have a special skybox background.
For this enter
loadsky "skyname"
//loadsky "ik2k/env/iklake"
//loadsky "socksy/mars"
where skyname is the path to the 6 required images for a skybox.
Check exisiting map CFGs for which skies are available and how to reference them exactly,
currently the comment lines in the above code example should work.

Other needs can be to have special music played, a different texture-set, mapmodels and last - but not least - the need to define trigger actions (for SP maps).
